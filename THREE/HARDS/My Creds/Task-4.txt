Agora que o invasor foi completamente expulso do
ambiente, que evidencia de atividade resta no ambiente

Step 1
Comece examinando os usuários do IAM que podem não pertencer ao
cliente com base na análise das duas tarefas anteriores. O que resta para
limpar do incidente

Oque pode ser:
1 - Ainda falta alguma coisa para remover do ambiente:
Procure mais afundo oque os usuários fizeram 
Procure por Lambda 
Procure por Logging ou ConsoleLogging
Procure por CloudTrail > Event History - Filter: UserName do usuario ou role name ou assumed role session name. Time range selecione o maximo possivel.
A resposta para este desafio esta no Cloudtrail

2 - Procure também por eventos das Access key, filtre pela access key para encontrar as coisas que elas fizeram.

Unsual Acitivty in AWS Accounts:

- Go to Cloudtrail
- Select the determined Trail
- Enable Insights on this trail


Pode demorar até 36 horas para o Insights aparecer.
